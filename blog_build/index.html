# 关于博客搭建

# vuepress 搭建个人博客

  1. 全局安装 vuepress
  • npm install -g vuepress
  1. 建立一根个文件夹
  • mkdir lqBlog
  1. 创建文件夹和文件 参考下面的目录 .
- ├─ docs
- │ ├─ README.md  配置首页信息
- │ └─ .vuepress
- │     └─ config.js 配置路由导航
- └─ package.json
  1. 配置 package.json 的命令
  • 生成 package.json 文件
npm init -y
  • 配置 package.json
 "dev": "vuepress dev docs",
"build": "vuepress build docs"
  1. 建立静态资源目录 .vuepress/public 文件夹来存放图片
  1. 配置默认主体 docs/README.md 将下面代码写到文件中
---
home: true
heroImage: /hero.png
actionText: Get Started →
actionLink: /guide/
features:
  - title: Simplicity First
    details: Minimal setup with markdown-centered project structure helps you focus on writing.
  - title: Vue-Powered
    details: Enjoy the dev experience of Vue + webpack, use Vue components in markdown, and develop custom themes with Vue.
  - title: Performant
    details: VuePress generates pre-rendered static HTML for each page, and runs as an SPA once a page is loaded.
footer: MIT Licensed | Copyright © 2018-present Evan You
---
  1. 在 docs/.vuepress/config.js
module.exports = {
  title: 'Hello VuePress',
  description: 'Just playing around'
}

npm run dev 运行博客

  1. 配置导航 .vuepress/config
themeConfig: {
  nav: [
    { text: '首页', link: '/' },
    { text: 'css', link: '/css/' }, //导航
    {
      //可以下拉导航
      text: 'js',
      items: [
        { text: 'JS基础', link: '/js/' },
        { text: 'js高级', link: '/jsg/' }
      ]
    }
  ]
}
  1. 让导航使用 导航所指文件夹下的 README.md 进行配置
  • 例如:docs/web/README.md进行配置
  1. 下拉导航的配置
themeConfig: {
  nav: [
    { text: '首页', link: '/' },
    { text: 'css', link: '/css/' }, //导航
    {
      //可以下拉导航
      text: 'js',
      items: [
        { text: 'JS基础', link: '/js/' },
        { text: 'js高级', link: '/jsg/' }
      ]
    }
  ]
}
  1. 简易配置侧边栏
sidebar: ['/', '/css/', '/js/', '/jsg/', '/vue/'] //侧边栏
  1. 侧边栏的分组
sidebar: [
  //多级侧边栏
  {
    title: 'vue',
    collapsable: true, // 是否折叠侧边栏
    children: ['/', '/props/']
  }
]
  • 详细博客结构配置见 vuePress 官网
  1. 发布到 GitHub

在文件根目录下建立 deploy.sh 文件 放以下代码

#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'

# 如果发布到 https://<USERNAME>.github.io
 git push -f git@github.com:lq110/lq110.github.io.git master

# 如果发布到 https://<USERNAME>.github.io/<REPO>
# git push -f git@github.com:<USERNAME>/<REPO>.git master:gh-pages

cd -
  1. git 仓库起名必须用 github 名开头
  • 例如 用户名为:lq110 仓库名就设置为:lq110.github.io
  1. 发布时候的命令 在 git 中运行
npm run deploy
  • 如果在运行命令中没有秘钥 请参考安装密钥 github 秘钥生成
